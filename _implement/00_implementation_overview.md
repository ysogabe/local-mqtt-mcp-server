# MQTT MCP Server 実装計画全体概要

## 📋 プロジェクト概要

### 目的
Model Context Protocol (MCP) に準拠したMQTTサーバーの段階的実装により、Claude Desktop と MQTT システムの統合を実現します。

### 開発方針
- **テスト駆動開発（TDD）**: すべてのコンポーネントでテストファースト
- **段階的実装**: 4つのフェーズに分けた段階的な開発
- **独立実行可能**: 各タスクが他に依存せず完結
- **実用重視**: 実際のプロダクション環境での使用を想定

## 🏗️ アーキテクチャ概要

### システム構成
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Claude        │    │  MQTT MCP       │    │   MQTT          │
│   Desktop       │◄──►│  Server         │◄──►│   Brokers       │
│   (MCP Client)  │    │  (This Project) │    │   (External)    │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### 主要コンポーネント
1. **MCP Interface Service**: MCPプロトコル実装
2. **Connection Manager**: MQTT接続管理
3. **Message Handler**: メッセージ送受信処理
4. **Subscription Manager**: 購読管理
5. **Event Manager**: イベント通知システム

## 📝 開発ガイドライン

### コード品質基準
- **TypeScript**: 厳格な型チェック有効
- **ESLint**: 推奨ルール + カスタムルール適用
- **Prettier**: 自動フォーマット
- **テストカバレージ**: 90%以上
- **ドキュメント**: 全公開APIにJSDoc記述

### 命名規則
- **クラス**: PascalCase (`ConnectionManager`)
- **メソッド/変数**: camelCase (`publishMessage`)
- **定数**: UPPER_SNAKE_CASE (`DEFAULT_TIMEOUT`)
- **ファイル**: kebab-case (`connection-manager.ts`)
- **インターフェース**: PascalCase + `I`プレフィックス (`IConnectionConfig`)
